GPS Test Hook Tool
GPS Test — это инструмент на базе Frida для перехвата (hooking) и подмены данных GPS-провайдера на аппаратном уровне. Позволяет тестировать геолокационные функции приложения без физического перемещения устройства.

Предварительные требования
Перед началом убедитесь, что у вас установлены:

Android Studio (с настроенным эмулятором x86_64).

Python 3.x.

ADB (Android Debug Bridge).

Инструкция по развертыванию
1. Подготовка Frida
Скачайте актуальную версию frida-server для вашей архитектуры (например, frida-server-android-x86_64) с официальных релизов GitHub.

Распакуйте архив.

Установите клиентскую часть на компьютер через терминал:

Bash
pip install frida-tools
2. Подготовка эмулятора и передача сервера
Запустите эмулятор через Android Studio и выполните следующие команды в Bash (заменив User на ваше имя пользователя):

Bash
# Переход в папку с загруженным сервером
cd C:/Users/”User”/downloads

# Перенос файла на устройство
adb push frida-server-16.7.8-android-x86_64 /data/local/tmp

# Проверка наличия файла и прав (ищите приписку -rwxrwxrwx)
adb shell ls -l /data/local/tmp

# Если права на выполнение отсутствуют (нет символа 'x'), выполните:
adb shell chmod +x /data/local/tmp/frida-server-16.7.8-android-x86_64
3. Запуск сервера на устройстве
Запустите процесс сервера в фоновом режиме и убедитесь, что он активен:

Bash
# Запуск
adb shell "/data/local/tmp/frida-server-16.7.8-android-x86_64 &"

# Проверка активного процесса
adb shell ps | findstr frida
Использование
Для запуска подмены данных перейдите в папку с вашими JS-скриптами и выполните команду запуска Frida:

Bash
# Переход в рабочую директорию
cd C:/Users/”User”/desktop

# Запуск скрипта
# -U (USB/Emulator), -f (Package name), -l (Script file)
frida -U -f net.example_app -l script.js

Примечание
Убедитесь, что версия frida-server на эмуляторе совпадает с версией frida-tools на вашем ПК.

Для работы хуков приложение должно быть запущено через Frida (-f), чтобы перехват произошел на этапе инициализации.
